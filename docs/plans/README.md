# 计划文档索引

`docs/plans/` 用于承接阶段计划、总规划和后续整理路线，帮助仓库持续按统一方向收敛。

## 当前文档

- `docs/plans/update-plan.md`：当前仓库的总目标、阶段结论、目录职责与后续收敛方向

## 使用顺序

1. 先读 `docs/plans/update-plan.md`，确认当前阶段目标与目录职责
2. 再进入 `docs/architecture/` 查看具体实施方案和执行清单
3. 涉及治理边界时，补充查看 `docs/governance/README.md`
4. 涉及执行过程与公开更新时，补充查看 `docs/maintenance/README.md`

## 使用约定

- 阶段性规划、整理路线和仓库级方向说明优先放入 `docs/plans/`
- 已确定的架构实现细节不要回写到计划文档，改放入 `docs/architecture/`
- 已执行完成的过程记录不要堆到计划文档，改放入 `docs/maintenance/`
