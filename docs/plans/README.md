# 计划文档索引

`docs/plans/` 用于承接阶段计划、总规划和后续整理路线，帮助仓库持续按统一方向收敛。

## 当前文档

- `docs/plans/update-plan.md`：当前仓库的总目标、阶段结论、目录职责与后续收敛方向
- `docs/plans/2026-04-02-Astro主链路收口与Hexo去除改造计划书.md`：Astro 主链路收口与 Hexo 去除改造计划
- `docs/plans/rainyun-topic-13009-integration-plan.md`：Rainyun 话题 13009 的项目化整合计划（模块拆分、阶段推进、验收标准）

## 使用顺序

1. 先读 `docs/plans/update-plan.md`，确认当前阶段目标与目录职责
2. 再读 `docs/plans/2026-04-02-Astro主链路收口与Hexo去除改造计划书.md`，确认主链路收口动作与验收标准
3. 再读 `docs/plans/rainyun-topic-13009-integration-plan.md`，确认专题整合范围与分阶段落地路径
4. 再进入 `docs/architecture/` 查看具体实施方案和执行清单
5. 涉及治理边界时，补充查看 `docs/governance/README.md`
6. 涉及执行过程与公开更新时，补充查看 `docs/maintenance/README.md`

## 使用约定

- 阶段性规划、整理路线和仓库级方向说明优先放入 `docs/plans/`
- 已确定的架构实现细节不要回写到计划文档，改放入 `docs/architecture/`
- 已执行完成的过程记录不要堆到计划文档，改放入 `docs/maintenance/`
- 计划文档命名统一采用 `YYYY-MM-DD-中文主题名.md`
- 计划文档内容建议至少包含：文档定位、目标、范围、执行项、验收标准、后续建议
