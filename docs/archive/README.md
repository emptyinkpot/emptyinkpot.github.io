## 使用顺序

1. 默认先查看 `docs/guides/` 中的现行版本
2. 只有在需要追溯历史内容差异时，才进入 `docs/archive/`
3. 如需确认仓库边界或整理原则，补充查看 `docs/plans/README.md` 与 `docs/governance/README.md`

## 使用约定

- 归档文件只保留历史参考价值，不应继续作为现行真源
- 新的现行说明应进入 `docs/guides/`、`docs/plans/`、`docs/governance/` 或 `docs/maintenance/`
- 如归档文档已经失去保留价值，应在明确确认后再统一处理
