# Anti CMS Rules

MyBlog borrows content ideas from CMS systems, but it must not collapse into a CMS collection website.

## Do Not Build

- category grid as homepage
- collection page as primary destination
- admin dashboard visual language for public reading
- table/card management UI as the reader
- generic "content list" as product identity

## Keep Instead

- mixed object runtime stream
- reader drawer
- collection lens
- object projection
- graph/search as runtime surfaces
- spatial continuity

## Reason

The project is a personal knowledge runtime. A CMS page hierarchy loses the object continuity that makes the system useful.
