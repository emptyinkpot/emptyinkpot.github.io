这里用来记录博客的持续维护与迭代。

每一条更新都会尽量回答三件事：

1. 更新了哪些内容
2. 涉及哪些技术或配置
3. 最终实现效果是什么

这样后续无论是写 GitHub 更新说明、回顾版本变化，还是整理站点演进过程，都会更清楚。

## 使用方式

后续追加日志时，建议沿用下面这个结构：

```md
## YYYY-MM-DD / 更新标题

### 更新内容
- ...

### 涉及技术
- ...

### 实现效果
- ...
```

- Hexo 首页生成逻辑

### 实现效果

- 首页阅读压力明显降低，浏览路径更接近正常博客站点

- “为什么本地改了但线上没变”这类常见误解被提前解释掉了
